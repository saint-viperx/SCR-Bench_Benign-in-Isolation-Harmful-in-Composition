# 演示文稿内容分析框架

在开始制作任何演示文稿之前，必须使用此框架分析用户提供的内容。

---

## 一、分析维度

### 1.1 内容理解

| 维度 | 问题 | 分析结果 |
|------|------|----------|
| **核心主题** | 这个演示文稿要传达什么？ | [填写] |
| **目标受众** | 谁会看这个演示？（投资人/客户/内部团队/学生） | [填写] |
| **使用场景** | 在什么场景下使用？（路演/汇报/培训/会议） | [填写] |
| **预计时长** | 演讲时间？（决定页数和信息密度） | [填写] |
| **关键信息** | 必须传达的 3-5 个核心观点是什么？ | [填写] |

### 1.2 结构映射

| 维度 | 建议 |
|------|------|
| **总页数** | 根据内容量和时长建议：5分钟约5-7页，10分钟约8-12页，20分钟约15-20页 |
| **章节划分** | 将内容分成 3-5 个逻辑章节 |
| **页面类型** | 每页的类型（封面/目录/内容/数据/总结） |
| **信息层级** | 哪些是核心信息，哪些是支撑信息 |

### 1.3 视觉机会

在分析内容时，识别以下可视化机会：

| 类型 | 适用场景 | 示例 |
|------|---------|------|
| **数据图表** | 数字、趋势、对比 | 柱状图、折线图、饼图 |
| **流程图** | 步骤、过程、工作流 | 时间线、流程箭头 |
| **对比表格** | 多项对比、特性矩阵 | 功能对比、方案对比 |
| **图标图示** | 概念、特性、要点 | 图标+文字说明 |
| **矩阵/象限** | 二维分类、定位 | 2x2 矩阵、四象限图 |
| **层级图** | 组织结构、分类 | 金字塔、树状图 |

### 1.4 设计方向

| 维度 | 选项 |
|------|------|
| **色彩基调** | 专业蓝 / 活力橙 / 科技紫 / 商务灰 / 自然绿 / 其他 |
| **风格倾向** | 简约 / 商务 / 科技 / 创意 / 学术 |
| **品牌要素** | 是否有品牌色/logo 需要融入？ |
| **视觉密度** | 简洁（大量留白）/ 标准 / 紧凑（信息密集） |

---

## 二、分析输出模板

完成分析后，输出 `analysis.md` 文件：

```markdown
# 演示文稿分析报告

## 基本信息
- **主题**: [演示文稿主题]
- **受众**: [目标受众]
- **场景**: [使用场景]
- **时长**: [预计时长]

## 核心信息
1. [关键信息1]
2. [关键信息2]
3. [关键信息3]

## 结构建议
- **总页数**: [建议页数]
- **章节划分**:
  1. [章节1]: [页数]
  2. [章节2]: [页数]
  3. [章节3]: [页数]

## 可视化机会
| 页面 | 内容 | 可视化类型 |
|------|------|-----------|
| P3 | [内容描述] | [图表类型] |
| P5 | [内容描述] | [表格/对比] |

## 设计方向
- **配色**: [配色方案]
- **风格**: [风格关键词]
- **品牌**: [品牌要求]
```

---

## 三、分析检查清单

在完成分析前，确认以下内容：

- [ ] 已明确核心主题和目标受众
- [ ] 已识别所有关键信息点
- [ ] 已建议合理的页面结构
- [ ] 已识别可视化机会
- [ ] 已确定设计方向
- [ ] 分析结果已保存到 `analysis.md`
