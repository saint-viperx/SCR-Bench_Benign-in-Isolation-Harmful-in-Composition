# Demo 验证工作流

在批量生成所有页面之前，通过 2-3 个关键页面的 Demo 验证设计方向和技术可行性。

---

## 一、目的

Demo 验证阶段的目标：

1. **验证设计方向** - 确保整体视觉风格符合用户预期
2. **验证技术可行性** - 确保复杂布局能正确转换为 PPTX
3. **建立设计模板** - Demo 页面作为后续批量生成的参考标准
4. **降低返工成本** - 提前发现问题，避免批量生成后大规模修改

---

## 二、页面选择标准

从大纲中选择 2-3 个代表性页面：

| 优先级 | 页面类型 | 选择原因 | 必选/推荐 |
|--------|---------|---------|----------|
| 1 | 封面 | 设定整体视觉基调、配色、字体 | **必选** |
| 2 | 最复杂内容页 | 验证复杂布局能力（多栏、图表、表格） | **必选** |
| 3 | 数据页（如有） | 验证图表/表格呈现效果 | 推荐 |
| 4 | 特殊页面 | 验证特殊设计元素（引用、时间线等） | 可选 |

### 选择示例

```
大纲共 10 页：
- P1: 封面 ← 必选（设定基调）
- P2: 目录
- P3: 行业背景（文字+数据）
- P4: 产品介绍（复杂双栏）← 必选（最复杂）
- P5: 技术架构（流程图）← 推荐（验证图示）
- P6: 市场数据（图表）
- P7: 团队介绍
- P8: 商业模式
- P9: 发展规划
- P10: 结尾

Demo 页面选择: P1, P4, P5
```

## 三、Demo 生成流程

### 3.1 流程图

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│ 1. 确定     │ → │ 2. 生成     │ → │ 3. 截图     │
│ Demo 页面   │    │ HTML        │    │ 展示        │
└─────────────┘    └─────────────┘    └─────────────┘
                                            │
       ┌────────────────────────────────────┘
       │
       v
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│ 4. 收集     │ → │ 5. 处理     │ → │ 6. 进入     │
│ 用户反馈    │    │ 反馈        │    │ Phase 2     │
└─────────────┘    └─────────────┘    └─────────────┘
```

### 3.2 详细步骤

#### 步骤 1: 确定 Demo 页面
- 根据大纲选择 2-3 个页面
- 告知用户选择理由
- 确认用户同意

#### 步骤 2: 生成 HTML
- 使用 Gemini CLI 生成 HTML（timeout: 180000）
- 严格遵循 html2pptx 约束
- 应用大纲中确定的设计规范

#### 步骤 3: 截图展示
- 使用 screenshot-html.cjs 生成截图
- 展示给用户查看
- 说明设计要点

#### 步骤 4: 收集用户反馈
- 使用 AskUserQuestion 收集意见
- 记录具体反馈到 `demo-feedback.md`

#### 步骤 5: 处理反馈
- 根据反馈类型决定下一步（见下表）
- 如需修改，重新生成 Demo

#### 步骤 6: 进入 Phase 2
- 用户确认满意后，进入批量生成阶段

---

## 四、反馈处理策略

| 反馈类型 | 示例 | 处理方式 |
|---------|------|---------|
| **整体风格不满意** | "太商务了，想要更有活力" | 返回 Phase 0.3 重新确认设计方向 |
| **配色问题** | "蓝色太深了" | 更新 design-spec，重新生成 Demo |
| **布局需要调整** | "希望图表更大" | 更新 outline.md，重新生成 Demo |
| **内容问题** | "这个要点描述不准确" | 更新 outline.md，重新生成 Demo |
| **细节问题** | "字号可以再大一点" | 记录到反馈文件，批量生成时注意 |
| **满意** | "很好，可以继续" | 进入 Phase 2 批量生成 |

## 六、退出条件

Demo 验证通过的标准：

- [ ] 封面风格用户确认满意
- [ ] 复杂页面布局技术可行
- [ ] 整体设计方向明确
- [ ] 用户明确同意进入批量生成

**重要**: 必须获得用户明确确认才能进入 Phase 2。不要假设用户满意。

---

## 七、使用 AskUserQuestion 确认

Demo 展示后，使用以下问题确认：

```
## Demo 验证

我已经生成了以下 Demo 页面供您预览：

**P1 封面**: [简述设计要点]
**P[X]**: [简述设计要点]

请问：
1. 整体视觉风格是否符合预期？
2. 配色和字体是否满意？
3. 布局和排版是否需要调整？
4. 是否可以开始批量生成所有页面？

如有任何需要调整的地方，请详细说明。
```
