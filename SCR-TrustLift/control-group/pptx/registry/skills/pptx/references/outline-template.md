# 演示文稿大纲模板

基于内容分析结果，使用此模板规划页面结构。

---

## 一、大纲文件结构

创建 `outline.md` 文件，包含以下内容：

```markdown
# [演示文稿标题] - 大纲

## 基本信息
- **标题**: [演示文稿标题]
- **总页数**: [建议页数]
- **预计时长**: [分钟]
- **目标受众**: [受众描述]

## 设计规范
- **主色**: [色号，如 #3182CE]
- **辅色**: [色号，如 #2D3748]
- **背景色**: [色号，如 #F7FAFC]
- **字体**: Arial, sans-serif
- **风格关键词**: [3-5 个关键词]

---

## 页面规划

### P1 - 封面
- **类型**: cover
- **标题**: [主标题]
- **副标题**: [副标题/slogan]
- **视觉描述**: [背景、图片、装饰元素描述]
- **备注**: [其他说明]

### P2 - 目录/概览
- **类型**: toc
- **章节列表**:
  1. [章节1名称]
  2. [章节2名称]
  3. [章节3名称]

### P3 - [页面标题]
- **类型**: content | data | comparison | process | quote
- **核心信息**: [这页要传达的关键点]
- **布局建议**: 单栏 | 双栏 | 图文 | 全图
- **内容详情**:
  - [要点1]
  - [要点2]
  - [要点3]
- **视觉元素**: [图表/图标/图片描述]

### P4 - [页面标题]
- **类型**: data
- **核心信息**: [关键数据点]
- **布局建议**: 双栏（左文右图）
- **数据内容**:
  | 指标 | 数值 |
  |------|------|
  | [指标1] | [数值1] |
  | [指标2] | [数值2] |
- **图表类型**: [柱状图/折线图/饼图]

... [更多页面]

### PN - 结尾
- **类型**: closing
- **内容**: [CTA/联系方式/致谢]
- **视觉描述**: [结尾页设计]
```

---

## 二、页面类型说明

| 类型 | 英文 | 用途 | 布局建议 |
|------|------|------|---------|
| 封面 | cover | 开场、主题展示 | 居中大标题 |
| 目录 | toc | 内容概览 | 列表或卡片 |
| 章节页 | section | 章节分隔 | 大字标题 |
| 内容页 | content | 文字说明 | 单栏或双栏 |
| 数据页 | data | 数据展示 | 图表为主 |
| 对比页 | comparison | 对比分析 | 表格或双栏 |
| 流程页 | process | 流程说明 | 时间线或步骤 |
| 引用页 | quote | 名言引用 | 居中大字 |
| 结尾页 | closing | 总结、CTA | 居中简洁 |

---

## 三、布局建议

### 3.1 单栏布局
- 适用：叙事性内容、重点强调
- 结构：标题 + 正文/要点列表

### 3.2 双栏布局
- 适用：图文并排、对比展示
- 结构：左文右图 或 左图右文
- 比例建议：40%/60% 或 50%/50%

### 3.3 多卡片布局
- 适用：多个并列概念、特性展示
- 结构：3-4 个等宽卡片
- 注意：每卡片内容要均衡

### 3.4 全图布局
- 适用：视觉冲击、情感表达
- 结构：背景图 + 叠加文字
- 注意：确保文字可读性

## 四、确认检查清单

大纲完成后，需与用户确认：

### 结构确认
- [ ] 总页数是否合适？
- [ ] 章节划分是否合理？
- [ ] 每页的核心信息是否准确？
- [ ] 信息的逻辑顺序是否正确？

### 设计确认
- [ ] 配色方案是否符合预期？
- [ ] 风格倾向是否正确？
- [ ] 是否有品牌要求需要融入？

### 优先级确认
- [ ] 哪些页面最重要？（用于 Demo 验证）
- [ ] 有没有特别复杂的页面需要特别关注？

---

## 五、使用 AskUserQuestion 确认

使用以下问题与用户确认大纲：

```
## 大纲确认

请确认以下内容是否符合您的预期：

**1. 页面结构**
- 总共 [N] 页，分为 [M] 个章节
- [简述各章节内容]

**2. 设计方向**
- 配色：[描述]
- 风格：[描述]

**3. Demo 页面选择**
建议先生成以下页面作为 Demo 验证：
- P1 封面（设定整体视觉基调）
- P[X]（最复杂的内容页）

请问：
1. 大纲结构是否需要调整？
2. 设计方向是否正确？
3. 是否同意先制作 Demo 验证？
```
